# ST-003: 검색 인터랙션 & 무한 스크롤 결과 화면
- 분류: Triage
- 긴급도: 중간
- 제안자: 미정
- 목표 릴리스: 미정

## 배경
- 기존 검색은 별도 페이지로 전환해야 해서 흐름이 끊긴다.
- 헤더 검색 아이콘을 눌렀을 때 인라인 입력칸이 바로 열리고, Velog처럼 상단에 검색창과 결과 개수, 아래로 세로 리스트 형태의 카드가 나열되는 경험이 요구된다.
- 검색 결과 카드는 가로폭을 넉넉하게 사용하며 썸네일, 제목, 태그, 작성자 정보, 요약, 메타데이터를 노출한다. 스크롤이 길어지므로 무한 스크롤과 상태 처리가 필수다.

## 완료 조건
- [ ] 헤더 검색 아이콘 클릭 시 동일 페이지 상단에 검색 입력 박스가 애니메이션으로 확장되고 자동 포커스가 된다.
- [ ] 검색 입력에 디바운스(예: 500ms) 로직이 적용되어 입력 중단 후 자동 검색이 실행되며, 엔터 입력 시 즉시 검색이 실행된다.
- [ ] 검색 결과 영역 상단에 `총 N개의 포스트를 찾았습니다.` 형식의 결과 개수가 표시된다.
- [ ] 결과 카드는 1열 리스트 형태로 썸네일(좌/상단), 제목, 태그, 요약, 작성자 아바타/닉네임, 작성일, 댓글 수, 좋아요 수가 순서 있게 배치된다.
- [ ] 카드 썸네일 비율, 텍스트 줄수 제한, 긴 제목/요약에 대한 말줄임 처리가 적용되어 시각적 일관성이 유지된다.
- [ ] 무한 스크롤(Intersection Observer 등)로 추가 데이터를 로드하며, 로딩 스피너/스켈레톤, 더 이상 결과 없음을 알리는 종료 메시지가 제공된다.
- [ ] 검색 결과 없음, 에러, 네트워크 지연 등 예외 상태 메시지와 재시도 버튼이 정의되고 구현된다.
- [ ] 검색 성공 시 URL 쿼리 스트링으로 검색어와 정렬 옵션이 반영되어 새로고침 후에도 동일한 상태가 유지된다.
- [ ] 검색 API는 검색어, 페이지/limit, 정렬 옵션을 지원하고, 각 결과에 썸네일·요약·태그·메타데이터를 포함한다.

## 작업
- UX/UI
  - 검색 입력 확장 모션, 결과 개수 표시 영역, 1열 리스트 카드 레이아웃 시안 제작.
  - 썸네일 크기, 텍스트 스타일, 메타 정보 배치, 태그 배지 스타일 정의.
  - 로딩/결과 없음/에러 상태 및 무한 스크롤 종료 영역 디자인.
  - 반응형 가이드: 데스크톱/태블릿/모바일에서 카드 폭·정렬·여백 규칙 문서화.
- 프론트엔드
  - 헤더 검색 상태 토글, 디바운스 로직, 자동 포커스 처리 구현.
  - 검색 결과 컴포넌트 작성: 1열 리스트 카드 렌더링, 결과 개수 표시, 무한 스크롤 구현.
  - Intersection Observer 활용 시 옵션 튜닝 및 폴백 처리, URL 쿼리 동기화.
  - 썸네일 없는 경우 기본 이미지 적용, 긴 텍스트 말줄임, 접근성(ARIA, 포커스 이동) 검증.
- 백엔드/API
  - 검색 API의 페이지네이션/정렬 옵션 및 응답 구조 점검, 누락 필드(썸네일/태그/카운트) 추가.
  - 대량 데이터 조회 시 성능 검토 및 캐싱/쿼리 최적화, 레이트리밋 정책 정비.
  - API 문서 업데이트 및 프론트엔드와 연계 테스트 계획 수립.
- QA/테스트
  - 디바운스/엔터 검색, 무한 스크롤, 뒤로가기·새로고침 시나리오 테스트 케이스 작성.
  - 검색 결과 없음, API 오류, 타임아웃 등 엣지 케이스 검증.
  - 시각적 회귀 테스트(주요 브레이크포인트) 및 접근성 키보드 탐색 검증.
